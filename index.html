JS的跨域问题

JS由于安全方面的原因（同源策略），不允许发送ajax请求本域名以外的数据，例如www.a.com是不能访问到www.b.com的(c.a.com也不行)，统一域名不同端口也不行，即使两个域名指向同一个IP地址也不行。但是在某些大项目开发中，不同域名下的项目需要互相通信，因此跨域问题是有必要解决的。前辈们总结了很多方法，下面我们看几个常用的：

flash

使用flash是跨域解决的，但flash现在在web开发中渐渐淡出我们的视野了（性能及交互的不足）。

服务器代理

服务端同学可以做到跨域，但是这样会增加服务器的负担。

html5

html5跨域做到跨域，具体方法暂待补充。。。

document.domain

通过设置主域名与子域名的document.domain达到跨域的目的，比如说都设置为document.domain = ‘a.com’。

JSONP

JSONP方式是常用的跨域方式，JSONP = JSON + Padding,意思是将json数据作为内填充的形式使用。首先，我们知道，script标签的src属性引用加载js文件是没有跨域问题的，那么，我们不是可以将需要的数据放在js文件里，这样就取得到了。但是加载过来的js文件会立即执行，我们无法获取到它的内容，因此我们给想要的数据包裹一个函数，将数据作为参数传递进去,让它立即执行，而我们只需要在之前定义这个函数，然后告诉服务端函数名就行了。

__proto__与prototype

在研究this用法的时候会用构造函数举例说明，而在涉及到构造函数的时候基本都会提到prototype来解释原型链的关系；我在调试的时候经常发现对象里面有个__proto__的属性，于是百度谷歌这两者之间的关系，试图了解它们多一点。

new操作符到底做了什么？

  1. var me = {}，即创建了一个对象。
  2. me.__proto__ = Person.prototype。
  3. Person.call(me)，也就是说构造me。

关键是能否验证第二步？

其实每一个对象都会有自动初始化__proto__属性，当我们访问一个对象属性时，如果这个对象没有这个属性，js就会在它的__proto__属性对象里面去找这个属性，如果还没找到，则会继续在__proto__对象的__proto__属性对象里面找，直到某一层的__proto__属性为null。
这也是我们经常所说的原型链的概念。

查找salary属性,本身没有，去__proto__（me对象）找，找到已被修改过的salary属性。因此he.salary结果输出为：10000。

最后，我发现，prototype只存在于构造函数（Object，Array，Function，自定义的等等），实际的对象并没有这个属性，意思就是说所以对象和函数都会有__proto__属性，而只有函数会有prototype属性，值为一个空函数，而且基本是在new一个构造函数的时候会用到，原型链的本质还是在__proto__。

函数里的this

上面代码首先定义了一个函数，函数里面用到了this；直接执行函数，随后执行window的showPerson()方法，可以发现，打印出的是window对象；使用new操作符实例化函数之后，this代表的是person对象，我们一般把构造函数函数名首字母大写以和普通函数区分。

new操作符的作用

  1. 创建一个新的对象；
  2. 将构造函数的作用域赋给新对象（因此this就指向了这个新对象）；
  3. 执行构造函数中的代码（this.xxx就为这个新对象添加某些属性、方法）；
  4. 返回新的对象。

call和apply

这两个方法经常在关于prototype原型的时候看到，但现在咱们先不管原型，单说这两个方法，我把它们理解成扩展、继承的意思；它们作用相同，不同的是传参类型区别：apply方法只能传递数组作为参数，call方法则是都可以。

用CSS实现块的垂直居中

块的水平居中可以设置margin:auto；行内元素可以设置text-align:text-center，垂直居中可以使用vertical-align:middle。
块的垂直居中可以使用js监听window的resize事件来动态实现，但如果是做弹出层这种类型的弹窗，我们完全可以使用css来完成。

利用定位实现

把弹出层设置为绝对定位或者固定定位，距离顶端50%，上边距设置为负的本身高度，距离左端50%，左边距设置为负的本身宽度，即可实现。

利用伪元素和行内块的垂直居中属性实现

用一个容器把弹出层装起来，然后把弹出层设置为行内块元素，再添加一个同样为行内块属性并且宽度为0高度为100%容器高度的伪元素设置vertical-align为middle即可实现。
